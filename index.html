什么是 mlvpn

mlvpn 是一款类似于 OpenVPN 的软件，能够在两台计算机之间创建网络隧道。

mlvpn 封装网络数据包，使用 UDP 协议并加密后通过互联网发送到另一个位置。

mlvpn 的主要用途是创建绑定的/聚合的网络链接（链路聚合），以便利用多条链路的带宽。

此外，mlvpn 也可以作为普通的安全隧道守护进程使用，能够处理故障转移场景。

功能特性

  • 多条互联网连接的带宽聚合
  • 自动故障转移，无需更改 IP 地址或中断 TCP 连接
  • 使用 libsodium 进行连接加密和认证
  • 热配置重载（通过发送 SIGHUP 信号）
  • 可脚本化监控
  • 通过 UNIX 套接字或 TCP/HTTP 套接字进行远程监控（JSON API）

局限性

（暂无已知局限性文档）

非对等链路（3G/4G 与 *DSL 或 WIFI 与 *DSL）

如果启用了重排序功能，mlvpn 可以聚合差异很大的链路。

如果您有一条高延迟的 3G/4G 链路和一条 DSL 连接，请将 reorder_buffer_size 调整到合理值以获得良好性能。

请注意，创建的聚合链路将具有所有链路中最差的延迟，即 3G/4G 链路的延迟。

重排序很重要，因为数据包在每条路径上的发送速度不同。数据包会乱序到达，这会严重影响 TCP 性能。如果不启用重排序，您将只能获得最慢链路的带宽。

配置

mlvpn 为每条隧道使用两个配置文件。

mlvpn.conf

mlvpn.conf(5) 是一个 INI 风格的配置文件。用于设置接口名称、密钥、多条链路的网络配置以及第二个配置脚本的路径。

提示：使用以下命令查看手册页：man mlvpn.conf

mlvpn_updown.sh

mlvpn_updown.sh 是 mlvpn 在状态发生变化时调用的脚本。

例如，当 mlvpn 启动且链路被激活时，mlvpn_updown.sh 会被调用以启动隧道设备并准备通信。

使用 ps 检查 mlvpn 状态

您可以随时使用标准的 Unix 命令 ps 来检查 mlvpn 的运行状态。

mlvpn 会生成两个进程。一个是以 root 权限运行的特权进程，其名称中包含 [priv]。 另一个是以您选择的用户身份运行的进程（通过 mlvpn --user 参数指定）。

示例：

root 30222 30221 0 23:17 pts/8 00:00:00 mlvpn: adsl3g [priv]
ed  30223 30222 0 23:17 pts/8 00:00:00 mlvpn: adsl3g !3g @adsl

此输出表示隧道 3g 已断开，而 adsl 已连接。

隧道前缀说明

前缀含义
!已断开
@已连接且运行中
~已连接但丢包严重（超过配置的阈值）

热重载 mlvpn 配置

mlvpn 支持热配置重载。您可以通过向任意进程发送 SIGHUP 信号来重新加载配置。 

kill -HUP $(pidof mlvpn) # 或者使用 pkill -HUP mlvpn
警告：热重载配置会强制所有已建立的链路断开并重新连接。

mlvpn 中的安全性

安全性是 mlvpn 项目的重中之重。

mlvpn 尽最大努力保护您的系统和数据，采用权限分离、强密码学和持续集成。

权限分离

mlvpn 需要访问内核来创建隧道设备并配置该设备（需要 root 权限）。

为了限制特权范围，mlvpn 会生成一个名为 [priv] 的小进程，然后再生成另一个进程。

priv 进程只执行需要特权的任务：

  • 隧道接口创建
  • 隧道接口配置
  • 打开 mlvpn 配置文件
  • 名称解析（需要在 chroot 之外运行）

另一个进程以非特权用户身份运行（通常是 _mlvpn 或 mlvpn），在 chroot 环境中运行。它处理所有对外暴露的繁重工作。

这与 OpenSSH 使用的技术相同。

密码学

mlvpn 使用 libsodium 满足所有密码学需求。

特别是，mlvpn 使用密钥认证加密。

密码学用于两个目的：

  • 认证
  • 数据保护

认证

mlvpn 使用一个非常简单的协议来确保只有双方共享相同密钥时才会通信。

密钥以明文形式存储在 mlvpn 配置文件中。

此配置文件必须由 root 拥有，并设置权限为 0600，以防止其他用户读取/写入。

mlvpn 发送的每个控制数据包都会被双方加密和认证。

认证使用 Poly1305 MAC 完成。

加密

数据包也可以加密/认证，但可以通过 cleartext_data 配置标志禁用。

如果您的 CPU 资源有限，这会很有用。

如果您的数据反正要发送到互联网，使用 mlvpn 再加密一次没有太大意义。（加密 SSH 或 HTTPS 连接不会带来太多收益）

加密使用 XSalsa20 算法。

Linux 双 ADSL 上行链路聚合与故障转移

简介

本简短指南将帮助您配置 Linux 的策略路由，以便同时自动使用两条 ADSL 链路。

示例场景

        +------------------------------------------+
        |           MLVPN 服务器                    |
        |       128.128.128.128/32                 |
        |     100Mbps 快速互联网                   |
        +----------+-----------+-------------------+
                   |           |
                   |           |
           +-------+---+ +----+--------+
           |  ADSL 1   | |   ADSL 2    |
           | 192.168.1.1| | 192.168.2.1|
           +------+----+ +-----+------+
                  |              |
           +------+----+ +-----+------+
           | MLVPN 客户端| | MLVPN 客户端|
           | 路由器      | | 路由器      |
           +------+----+ +-----+------+
                  |              |
                  +------+------+
                         |
                  +------+------+
                  |   局域网    |
                  |192.168.0.0/24|
                  +-------------+

在此配置中，我们有以下设备：

  • MLVPN 服务器：拥有快速互联网连接（100Mbps），公网 IP 地址：128.128.128.128/32
  • ADSL 1 路由器：本地 IP 地址 192.168.1.1/24
  • ADSL 2 路由器：本地 IP 地址 192.168.2.1/24
  • 局域网：192.168.0.0/24
  • MLVPN 客户端路由器：多个私有 IP 地址

测试基本配置

查看当前路由表：

ip route show

测试到服务器的连通性：

ping -c 3 192.168.1.1
ping -c 3 192.168.2.1
ping -c 3 128.128.128.128

配置源路由

概念

源路由意味着内核不仅根据数据包的目标地址，还会根据数据包的来源地址来决定路由。

在我们的场景中，从 MLVPN 客户端发出的数据包，源地址可能是 192.168.1.x（ADSL 1）或 192.168.2.x（ADSL 2）。 源路由确保从 ADSL 1 发出的数据包通过 ADSL 1 的网关返回，从 ADSL 2 发出的数据包通过 ADSL 2 的网关返回。

开始配置

首先，定义路由表名称。编辑 /etc/iproute2/rt_tables：

# echo "1 adsl1" >> /etc/iproute2/rt_tables
# echo "2 adsl2" >> /etc/iproute2/rt_tables

为每个 ADSL 连接配置路由表：

# ADSL 1 路由表
ip route add default via 192.168.1.1 dev eth1 table adsl1
ip rule add from 192.168.1.0/24 table adsl1

# ADSL 2 路由表
ip route add default via 192.168.2.1 dev eth2 table adsl2
ip rule add from 192.168.2.0/24 table adsl2

测试

ip rule list
ip route show table adsl1
ip route show table adsl2

开机自动启动脚本

创建 /usr/local/sbin/source_routing：

#!/bin/bash

# 定义变量
ADSL1_GW="192.168.1.1"
ADSL2_GW="192.168.2.1"
ADSL1_IF="eth1"
ADSL2_IF="eth2"
ADSL1_NET="192.168.1.0/24"
ADSL2_NET="192.168.2.0/24"

# 清空旧规则
ip rule flush
ip route flush table adsl1
ip route flush table adsl2

# 添加路由表
ip route add default via $ADSL1_GW dev $ADSL1_IF table adsl1
ip route add default via $ADSL2_GW dev $ADSL2_IF table adsl2

# 添加策略规则
ip rule add from $ADSL1_NET table adsl1
ip rule add from $ADSL2_NET table adsl2

# 刷新路由缓存
ip route flush cache

echo "源路由配置完成"

设置执行权限并链接到启动目录：

chmod +x /usr/local/sbin/source_routing

配置 MLVPN

客户端配置

mlvpn0.conf：

[general]
name = mlvpn0
tunnel_password = "your_secure_password_here"
password = "your_client_password_here"
debug = 0

[adsl1]
host = 128.128.128.128
bindhost = 192.168.1.2
bindport = 65530

[adsl2]
host = 128.128.128.128
bindhost = 192.168.2.2
bindport = 65531

mlvpn0_updown.sh：

#!/bin/bash
# MLVPN 状态变化回调脚本
# 被 mlvpn 调用，参数为隧道名称和状态

TUNNEL=$1
STATUS=$2

if [ "$STATUS" = "up" ]; then
    echo "隧道 $TUNNEL 已激活"
    # 启动隧道接口
    ifconfig $TUNNEL 10.0.0.2 netmask 255.255.255.252 pointopoint 10.0.0.1 up
    # 添加路由
    ip route add 10.0.0.0/30 dev $TUNNEL 2>/dev/null
    # 重新加载源路由配置
    /usr/local/sbin/source_routing
elif [ "$STATUS" = "down" ]; then
    echo "隧道 $TUNNEL 已断开"
fi

服务端配置

mlvpn0.conf：

[general]
name = mlvpn0
tunnel_password = "your_secure_password_here"
mode = server
listen = 0.0.0.0:65530
debug = 0

# 管理接口
[status]
bindhost = 127.0.0.1
bindport = 4001

mlvpn0_updown.sh：

#!/bin/bash
# MLVPN 服务端状态变化回调脚本

TUNNEL=$1
STATUS=$2

if [ "$STATUS" = "up" ]; then
    echo "客户端隧道 $TUNNEL 已连接"
    # 配置隧道接口
    ifconfig $TUNNEL 10.0.0.1 netmask 255.255.255.252 pointopoint 10.0.0.2 up
    # 启用 IP 转发
    echo 1 > /proc/sys/net/ipv4/ip_forward
    # 添加 NAT 规则
    iptables -t nat -A POSTROUTING -s 10.0.0.0/30 -o eth0 -j MASQUERADE
    iptables -A FORWARD -i $TUNNEL -o eth0 -j ACCEPT
    iptables -A FORWARD -i eth0 -o $TUNNEL -m state --state RELATED,ESTABLISHED -j ACCEPT
elif [ "$STATUS" = "down" ]; then
    echo "客户端隧道 $TUNNEL 已断开"
    # 清理 NAT 规则
    iptables -t nat -D POSTROUTING -s 10.0.0.0/30 -o eth0 -j MASQUERADE 2>/dev/null
    iptables -D FORWARD -i $TUNNEL -o eth0 -j ACCEPT 2>/dev/null
    iptables -D FORWARD -i eth0 -o $TUNNEL -m state --state RELATED,ESTABLISHED -j ACCEPT 2>/dev/null
fi

过滤功能

mlvpn 中的过滤系统可以在聚合场景下使用。

某些协议会因数据包乱序或丢包而受到严重影响，例如 VoIP 系统。

为了解决这个问题，mlvpn 包含了一个叫做"过滤器"的系统。

在 mlvpn.conf 中，[filters] 段定义了匹配表达式的静态路径。

表达式使用标准的 BPF 表达式（与 tcpdump 或其他 libpcap 程序中使用的一样）。

过滤器对顺序敏感。

启用重排序与 VoIP 的 ADSL 和 SDSL

在此场景中，我们希望聚合所有协议的流量，除了 SIP UDP 端口 5060。

mlvpn.conf

[general]
name = mlvpn0
tunnel_password = "secure_password"
password = "link_password"
debug = 0
reorder_buffer_size = 64

[link_adsl]
host = remote.example.com
bindhost = 0.0.0.0
bindport = 65530

[link_sdsl]
host = remote.example.com
bindhost = 0.0.0.0
bindport = 65531

[filters]
# 将 SIP UDP 5060 流量固定到 SDSL 链路
sdsl = udp port 5060
# 其余流量可使用任一链路（默认）

说明

此配置在所有链路正常时，会将匹配 "udp port 5060" BPF 表达式的数据包仅发送到 sdsl 链路。

如果 sdsl 链路不可用，则会选择第二个匹配的接口。

过滤器采用排他性首匹配机制。这意味着如果数据包匹配某个表达式，且该接口已准备好接收数据，过滤将停止，数据包将被发送。

在该场景中，顺序非常重要，以便您选择首选路径。

BPF 表达式示例

表达式说明
udp port 5060匹配 SIP 协议（UDP 5060 端口）
tcp port 80匹配 HTTP 流量
host 10.0.0.1匹配特定主机的流量
net 192.168.0.0/16匹配特定网段的流量
udp and portrange 10000-20000匹配指定范围的 UDP 端口
提示：您可以在命令行中使用 tcpdump -i any -d <表达式> 来验证 BPF 表达式是否正确。

MLVPN 多条宽带聚合与中转机方案

核心原理

MLVPN 通过创建多条独立隧道（rtun）来聚合带宽。每条宽带对应一条隧道，所有隧道的数据汇聚到同一个虚拟接口上。

源码确认的关键特性

特性说明源码依据
隧道数量无上限理论上可以添加任意数量的隧道段src/mlvpn.c 中使用链表 rtunhead 管理隧道
bindhost 非必需客户端模式下 bindhost 默认为 NULL，自动绑定 0.0.0.0src/config.c 中 _conf_set_str_from_conf(... "bindhost" ... NULL)
每个隧道完全独立每条隧道有独立的 remotehost、remoteport、bindhost、bindportsrc/config.c 中逐段解析各隧道配置
负载均衡算法使用加权轮询（Weighted Round-Robin），基于 bandwidth_upload 权重src/mlvpn.c 中 mlvpn_rtun_recalc_weight()
故障转移机制单条隧道断开不影响其他隧道，系统自动从聚合中移除故障链路src/mlvpn.c 中 mlvpn_rtun_status_down()

方案一：直连多条宽带

网络拓扑

MLVPN 客户端
├── 宽带1 (192.168.1.x) ──> MLVPN 服务器:5080
├── 宽带2 (192.168.2.x) ──> MLVPN 服务器:5081
└── 宽带3 (192.168.3.x) ──> MLVPN 服务器:5082

客户端配置

[general]
mode = "client"
interface_name = "mlvpn0"
password = "统一密码"
reorder_buffer_size = 64
loss_tolerence = 50
statuscommand = "/etc/mlvpn/mlvpn_updown.sh"

# 宽带1
[broadband1]
bindhost = "192.168.1.100"    # 可选，明确指定出口
remotehost = "服务器公网IP"
remoteport = 5080
bandwidth_upload = 10240       # 10Mbps 上行

# 宽带2
[broadband2]
bindhost = "192.168.2.100"    # 可选
remotehost = "服务器公网IP"
remoteport = 5081
bandwidth_upload = 20480       # 20Mbps 上行

# 宽带3
[broadband3]
bindhost = "192.168.3.100"    # 可选
remotehost = "服务器公网IP"
remoteport = 5082
bandwidth_upload = 51200       # 50Mbps 上行

服务端配置

[general]
mode = "server"
interface_name = "mlvpn0"
password = "同一密码"
reorder_buffer_size = 64
loss_tolerence = 50
statuscommand = "/etc/mlvpn/mlvpn_updown.sh"

[broadband1]
bindport = 5080

[broadband2]
bindport = 5081

[broadband3]
bindport = 5082

方案二：通过中转机聚合

场景说明

当你的服务器没有公网 IP，或者需要通过中转机优化路由时，可以使用两台中转机分别转发不同端口的流量。

网络拓扑

MLVPN 客户端
├── 宽带1 ──> 中转机A (公网IP-A:5080) ──> MLVPN 服务器:5080
└── 宽带2 ──> 中转机B (公网IP-B:5081) ──> MLVPN 服务器:5081
源码确认：remotehost 支持不同 IP
从 src/config.c 可以确认，每个隧道段的 remotehost 和 remoteport 是独立解析的。客户端模式下，remotehost 是必需参数，但可以为每个隧道配置不同的 IP。

客户端配置（中转机方案）

[general]
mode = "client"
interface_name = "mlvpn0"
password = "统一密码"
reorder_buffer_size = 64
loss_tolerence = 50
statuscommand = "/etc/mlvpn/mlvpn_updown.sh"

# 隧道1 → 走中转机A
[tunnel_via_A]
bindhost = "192.168.1.100"    # 宽带1的出口IP
remotehost = "中转机A的公网IP"  # ← 不同的中转机IP
remoteport = 5080

# 隧道2 → 走中转机B
[tunnel_via_B]
bindhost = "192.168.2.100"    # 宽带2的出口IP
remotehost = "中转机B的公网IP"  # ← 不同的中转机IP
remoteport = 5081

服务端配置

[general]
mode = "server"
interface_name = "mlvpn0"
password = "同一密码"
reorder_buffer_size = 64
loss_tolerence = 50
statuscommand = "/etc/mlvpn/mlvpn_updown.sh"

[tunnel_via_A]
bindport = 5080

[tunnel_via_B]
bindport = 5081

中转机配置

中转机A 的 iptables 规则：

# 将 5080 端口的流量转发到 MLVPN 服务器
iptables -t nat -A PREROUTING -p udp --dport 5080 -j DNAT --to-destination MLVPN服务器IP:5080
iptables -t nat -A POSTROUTING -p udp -d MLVPN服务器IP --dport 5080 -j MASQUERADE

中转机B 的 iptables 规则：

# 将 5081 端口的流量转发到 MLVPN 服务器
iptables -t nat -A PREROUTING -p udp --dport 5081 -j DNAT --to-destination MLVPN服务器IP:5081
iptables -t nat -A POSTROUTING -p udp -d MLVPN服务器IP --dport 5081 -j MASQUERADE

关键注意事项

1. bindhost 的作用

场景是否需要 bindhost说明
单网卡不需要默认绑定 0.0.0.0
多网卡，每条宽带对应不同网卡建议填写确保流量从正确的网卡出去
所有宽带共用一个网关不需要系统路由会自动处理
源码确认：bindhost 在客户端模式下是可选的，默认为 NULL（即 0.0.0.0）。

2. remotehost 的独立性

源码确认：每个隧道段的 remotehost 和 remoteport 是完全独立的参数，可以为不同的隧道配置不同的目标 IP。

  • ✅ 隧道1 连接到中转机A（公网IP-A）
  • ✅ 隧道2 连接到中转机B（公网IP-B）
  • ✅ 两台中转机可以位于不同的地理位置
  • ✅ 流量会自动聚合到同一个虚拟接口

MLVPN 使用加权轮询（Weighted Round-Robin）算法分配流量：

配置示例：

[broadband1]
bandwidth_upload = 10240    # 10Mbps 上行

[broadband2]
bandwidth_upload = 20480    # 20Mbps 上行

[broadband3]
bandwidth_upload = 51200    # 50Mbps 上行

总带宽 = 10 + 20 + 50 = 80Mbps

4. 故障转移行为

当某条隧道断开时：

  1. MLVPN 检测到隧道超时（默认 60 秒）
  2. 将该隧道标记为 MLVPN_DOWN
  3. 从聚合池中移除该隧道
  4. 流量自动分配到剩余的健康隧道
  5. 如果隧道恢复，自动重新加入聚合池

状态前缀说明：

前缀含义
!已断开
@正常运行
~丢包严重（超过 loss_tolerence 阈值）

5. 中转机方案的前提条件

要实现真正的带宽聚合，必须满足：
  • ✅ 客户端的两条宽带必须能独立到达两台中转机
  • ✅ 流量不能走同一条线路（否则没有聚合效果）
  • ✅ 需要配置源路由（policy routing），确保每条宽带的流量从对应的网卡出去

源路由配置示例（客户端）：

# 创建路由表
echo "101 broadband1" >> /etc/iproute2/rt_tables
echo "102 broadband2" >> /etc/iproute2/rt_tables

# 配置路由
ip route add default via 192.168.1.1 dev eth1 table broadband1
ip route add default via 192.168.2.1 dev eth2 table broadband2

# 配置策略路由
ip rule add from 192.168.1.100 table broadband1
ip rule add from 192.168.2.100 table broadband2

重排序缓冲区

三、配置详解

MLVPN 每条隧道需要两个配置文件：

  • mlvpn.conf：主配置文件
  • mlvpn_updown.sh：状态变化回调脚本

3.1 mlvpn.conf 配置文件

语法说明

  • INI 风格，[section] 分段
  • # 开头为注释
  • 键值对用 = 分隔
  • 字符串用双引号包裹

[general] 全局配置段

[general]
# ===== 必需参数 =====

# 工作模式：server（服务端）或 client（客户端）
mode = "client"

# 隧道加密密码（客户端和服务端必须完全一致）
password = "你的安全密码"

# 状态变化回调脚本路径
statuscommand = "/etc/mlvpn/mlvpn_updown.sh"

# ===== 可选参数 =====

# 隧道接口名称（仅 Linux 有效）
interface_name = "mlvpn0"

# 隧道类型：tun（三层路由）或 tap（二层桥接）
tuntap = "tun"

# 超时时间（秒），心跳间隔 = timeout / 2
# 默认 30 秒，即每 15 秒发送一次心跳
timeout = 30

# 重排序缓冲区大小（数据包数）
# 用于处理乱序数据包，推荐值：64-128
# 设为 0 禁用重排序
reorder_buffer_size = 64

# 丢包容忍阈值（百分比）
# 链路丢包超过此值时标记为 lossy 并从聚合中移除
# 设为 100 或更大禁用丢包容忍
loss_tolerence = 50

# 明文数据模式（危险！仅调试用）
# 设为 1 时数据包不加密
cleartext_data = 0

# 延迟容忍阈值（毫秒）
# 链路延迟超过此值时标记为高延迟
# 设为 1000 或更大禁用
latency_tolerence = 0

隧道配置段

每个隧道需要独立的配置段，段名即隧道名称：

# 隧道 1 配置
[tunnel_name_1]
# 客户端参数
bindhost = "192.168.1.2"      # 绑定的本地地址
remotehost = "1.2.3.4"        # 服务端公网地址
remoteport = 5080             # 服务端端口

# 服务端参数（二选一）
bindhost = "0.0.0.0"          # 监听所有地址
bindport = 5080               # 监听端口

# 可选参数
bandwidth_upload = 61440      # 上传带宽限制（字节/秒）
timeout = 25                  # 覆盖全局超时设置
fallback_only = 0             # 是否仅作为故障转移链路

隧道参数详解：

参数必需说明
bindhost否绑定地址（IPv4）
bindport服务端必需监听端口
remotehost客户端必需远程服务器地址
remoteport客户端必需远程服务器端口
bandwidth_upload否最大上传带宽（字节/秒），0 为不限制
timeout否覆盖全局 timeout
fallback_only否1 = 仅在所有其他隧道断开时使用
bindfib否FIB 路由表（仅 FreeBSD/OpenBSD）

[filters] 过滤配置段

[filters]
# BPF 表达式语法（与 tcpdump 相同）
# 将 SIP 流量路由到 sdsl 链路
sdsl = udp port 5060
# 备用：如果 sdsl 不可用，走 adsl
adsl = udp port 5060
过滤器对顺序敏感，采用排他性首匹配机制。

3.3 mlvpn_updown.sh 回调脚本

当隧道状态变化时，MLVPN 会调用此脚本。

调用参数：

参数说明
$1隧道接口名称
$2状态名称
$3隧道名称（仅 rtun_up/down 时有值）

状态类型：

状态触发时机
tuntap_up至少一条隧道上线
tuntap_down所有隧道断开
rtun_up某条远程隧道上线
rtun_down某条远程隧道断开

模板脚本：

#!/bin/bash
error=0
trap "error=$((error|1))" ERR

tuntap_intf="$1"    # 接口名称
newstatus="$2"      # 状态名称
rtun="$3"           # 隧道名称

[ -z "$newstatus" ] && exit 1

(
  if [ "$newstatus" = "tuntap_up" ]; then
      echo "$(date) $tuntap_intf 隧道已启动"
      # 在此添加隧道启动后的配置命令
      # 例如：设置 MTU、添加路由等

  elif [ "$newstatus" = "tuntap_down" ]; then
      echo "$(date) $tuntap_intf 隧道已关闭"
      # 在此添加隧道关闭后的清理命令

  elif [ "$newstatus" = "rtun_up" ]; then
      echo "$(date) 远程隧道 [${rtun}] 已上线"

  elif [ "$newstatus" = "rtun_down" ]; then
      echo "$(date) 远程隧道 [${rtun}] 已断开"
  fi
) >> /var/log/mlvpn_updown.log 2>&1

exit $errors

设置权限：

chmod 700 /etc/mlvpn/mlvpn_updown.sh
chown root:root /etc/mlvpn/mlvpn_updown.sh

4.2 使用 systemd 管理服务（推荐）

创建 systemd 服务文件

tee /etc/systemd/system/mlvpn.service > /dev/null << 'EOF'
[Unit]
Description=MLVPN - Multi-Link Virtual Public Network
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=/usr/local/bin/mlvpn --user mlvpn -c /etc/mlvpn/mlvpn.conf
Restart=on-failure
RestartSec=5
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
EOF

# Linux (iptables)

问题 1：mlvpn 无法启动

症状：执行 mlvpn 命令后立即退出

/etc/mlvpn/mlvpn_updown.sh（服务端）：

#!/bin/bash
error=0
trap "error=$((error|1))" ERR

tuntap_intf="$1"
newstatus="$2"
rtun="$3"

[ -z "$newstatus" ] && exit 1

(
if [ "$newstatus" = "tuntap_up" ]; then
    echo "$(date) $tuntap_intf 隧道已启动"
    /sbin/ip link set dev $tuntap_intf mtu 1400 up
    /sbin/ip route add 192.168.0.0/24 dev $tuntap_intf
    /sbin/iptables -t nat -A POSTROUTING -o eth0 -s 192.168.0.0/24 -j MASQUERADE
elif [ "$newstatus" = "tuntap_down" ]; then
    echo "$(date) $tuntap_intf 隧道已关闭"
    /sbin/iptables -t nat -D POSTROUTING -o eth0 -s 192.168.0.0/24 -j MASQUERADE
elif [ "$newstatus" = "rtun_up" ]; then
    echo "$(date) 远程隧道 [${rtun}] 已上线"
elif [ "$newstatus" = "rtun_down" ]; then
    echo "$(date) 远程隧道 [${rtun}] 已断开"
fi
) >> /var/log/mlvpn_updown.log 2>&1

exit $errors

7.3 安全最佳实践

  1. 密码安全：使用强密码（建议 16 位以上），客户端和服务端必须一致
  2. 文件权限：配置文件 600，回调脚本 700
  3. 运行用户：始终使用非 root 用户运行 MLVPN
  4. 防火墙：仅允许信任的 IP 连接隧道端口
  5. 加密：保持 cleartext_data = 0，不要禁用加密
  6. 监控：定期检查日志，及时发现异常

7.4 性能调优建议

场景建议参数
高延迟链路timeout = 60, reorder_buffer_size = 128
低带宽链路bandwidth_upload 限制
VoIP 应用启用过滤功能，将 SIP 流量指定到低延迟链路
负载均衡配置源路由，根据源 IP 分配链路
故障转移配置 fallback_only 参数
